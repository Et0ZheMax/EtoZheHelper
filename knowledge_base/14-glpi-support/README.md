# GLPI / Support

Раздел для заявок, коммуникации, диагностики и красивого закрытия обращений.

## Формула хорошего комментария

```text
Что сделал -> что выяснил -> что исправил -> как проверил -> текущий статус
```

## Не писать

- "готово" без деталей;
- "проблема была в системе";
- обвинять пользователя;
- писать внутренние догадки без фактов.

## Писать

- конкретно;
- спокойно;
- с проверкой результата;
- если нужна эскалация — указать куда и почему.
